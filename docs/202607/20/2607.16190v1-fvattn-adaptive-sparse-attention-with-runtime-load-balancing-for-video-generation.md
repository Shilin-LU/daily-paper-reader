## 摘要
视频扩散Transformer处理长时空序列，使得自注意力成为高分辨率视频生成的主要瓶颈。免训练稀疏注意力可降低此成本，但自适应Top-p路由在多GPU序列并行下会产生不均的每头工作负载。由此产生的工作负载异质性将稀疏注意力转变为秩级拖后腿问题。我们提出\method{}，一种免训练的稀疏注意力系统，旨在提升多GPU序列并行下自适应稀疏注意力的分布式执行效率。\method{}采用Top-p路由、Top-k安全下限和视频感知块组织作为稀疏路由前端，并在运行时修复物化掩码。运行时负载均衡通过点对点通信迁移少量重负载头，以缩短当前关键路径。松弛感知稀疏增强利用额外的价值块填充非关键秩的剩余松弛，而重叠将调度和迁移开销隐藏在现有计算之后。在步进蒸馏的Wan2.2 I2V上，\method{}将平均负载不平衡从1.34降至1.08，相比FlashAttention实现了4.41倍的注意力加速，同时达到2.02-2.11倍的DiT推理加速，并保持了有竞争力的视频质量。

## Abstract
Video Diffusion Transformers process long spatio-temporal sequences, making self-attention the main bottleneck in high-resolution video generation. Training-free sparse attention reduces this cost, but adaptive Top-$p$ routing creates uneven per-head workloads under multi-GPU sequence parallelism. The resulting workload heterogeneity turns sparse attention into a rank-level straggler problem. We present \method{}, a training-free sparse-attention system that improves the distributed execution efficiency of adaptive sparse attention under multi-GPU sequence parallelism. \method{} uses Top-$p$ routing, a Top-$k$ safety floor, and video-aware block organization as the sparse-routing frontend, then repairs the materialized mask at runtime. Runtime Load Balancing migrates a small number of heavy heads via P2P communication to shorten the current critical path. Slack-Aware Sparse Augmentation fills residual non-critical-rank slack with additional high-value blocks, while overlap hides scheduling and migration overhead behind existing computation. On step-distilled Wan2.2 I2V, \method{} reduces average load imbalance from 1.34 to 1.08 and delivers a $4.41\times$ attention speedup over FlashAttention, while achieving a $2.02$--$2.11\times$ DiT inference speedup with competitive video quality.

## 1. 核心问题与研究动机（背景）

- **背景**：视频扩散Transformer（Video DiTs）在视觉质量上取得显著进步，但推理成本极高。例如，Wan2.2-14B I2V生成5秒720p视频在单块NVIDIA H20上需要约两小时，其中自注意力占总推理时间的74.1%。虽然步进蒸馏能减少去噪步数（最多加速25×），但每步的序列长度和二次注意力成本不变，注意力仍是主要瓶颈。
- **问题**：免训练稀疏注意力通过Top-p路由自适应地保留少量关键块，提升了保真度，但在多GPU Ulysses式序列并行下，不同注意力头保留的块数差异巨大，导致头级工作负载不均。这种不均转化为秩级（rank-level）拖后腿问题：慢速GPU成为同步注意力的瓶颈，其他GPU空闲等待，部分抵消了稀疏带来的计算节省。
- **核心矛盾**：自适应稀疏（Top-p）的保真度优势与分布式执行效率之间的矛盾。现有预调度方案基于历史或近似信号，但在少步生成中步间变化极大（头密度最大变化97%，秩负载最大变化44%），预测不准确；全后置重分区则需大规模数据移动，开销过大。

## 2. 方法论：核心思想、关键技术细节与流程

### 2.1 核心思想
FVAttn是一种免训练的稀疏注意力系统，在自适应稀疏掩码物化后，通过**运行时负载均衡（Runtime Load Balancing, RLB）** 和**松弛感知稀疏增强（Slack-Aware Sparse Augmentation, SASA）** 两级机制，以轻量点对点通信和空闲计算填充的方式，在保持视频质量的同时提升多GPU效率。

### 2.2 前端：自适应稀疏路由
- 使用**Top-p路由**（基于累积概率阈值p动态决定保留块数）加上**Top-k安全底限**（防止过度稀疏化）。
- 视频感知块组织：沿Hilbert曲线重排块，使空间邻近令牌落入相邻块，提高路由相似性估计。
- 输出：每个头的实际工作负载\( L_h \)和块重要性排序。

### 2.3 运行时负载均衡（RLB）
- **目标**：在稀疏掩码物化后，通过有限点对点头部迁移缩短当前注意力阶段的关键路径。
- **观察**：负载不均高度局部化，通常只有少量重头导致大部分偏斜。
- **约束**：每秩最多迁移一个本地头（默认8 GPU下约20%头部），迁移计划表示为置换σ，使通信开销小。
- **优化目标**：最小化迁移后的最大秩负载和秩负载方差，搜索空间小（如枚举符合条件的置换），各GPU独立运行相同搜索（通过一次轻量All-Gather获取全局密度）。
- **效果**：平均负载不均衡因子从1.34降至1.08。

### 2.4 松弛感知稀疏增强（SASA）
- **目标**：利用RLB后非关键秩上的剩余松弛（空闲等待时间）额外计算高价值块，提高稀疏掩码覆盖率，但不延长全局关键路径。
- **步骤**：
  - 计算松弛\(\Delta L_r = L_{\max} - L_r\)。
  - 分配额外稀疏预算\( B_r = n_1 \Delta L_r \)（系数\( n_1=0.8 \)），从已排序的重要块中选取。
  - 触发阈值\( n_2 = 0.07 \)：仅当\(\Delta L_r > n_2 L_{\max}\)时才启用，避免微小松弛的开销。
- **非等价于全局提高Top-p**：全局提高会延长所有秩的关键路径；SASA只填充空闲秩。

### 2.5 高效实现：重叠与优化
- **CPU-GPU重叠**：CPU搜索RLB计划和计算SASA预算，同时GPU执行V量化，隐藏调度成本。
- **计算-通信重叠**：异步发起P2P迁移，同时GPU对非迁移头进行V量化和SASA增强，在消费迁移数据前同步。
- **总开销**：各组件（密度交换、计划搜索、P2P迁移、SASA增强等）的可见临界路径开销仅约0.7ms（占注意力延迟1.87%）。

### 2.6 分阶段启用机制
- 若负载不均衡因子\( \rho < 1.05\)：跳过额外机制。
- 若\( 1.05 \leq \rho < 1.10\)：仅启用SASA。
- 若\( \rho \geq 1.10\)：先RLB再SASA。

## 6. 主要结论与发现

- FVAttn在三个视频生成模型上**同时提升质量和效率**，实现“同速更高质量”或“同质更高速”。

## 7. 优点

1. **创新性**：提出运行时负载均衡的修复范式，区别于传统预测式重分区，直接基于物化后真实工作负载进行轻量修复。
2. **高效性**：仅迁移少量头部（每GPU最多一个）即可显著改善不均衡，通信开销小。
3. **质量-效率协同**：SASA将同步等待时间转化为高质量计算，实现“事半功倍”。
4. **系统实现**：通过CPU-GPU和计算-通信双重重叠，使运行时开销几乎不可见。
5. **通用性**：RLB和SASA可作为即插即用组件，适用于各种Top-p稀疏注意力前端。
6. **实验严谨**：覆盖多模型、多指标、多消融，与多个SOTA对比，公平性有保障。

## 8. 不足与局限

1. **适用范围**：主要针对长时空序列视频生成；对于短序列或弱稀疏性任务（如图像生成），稀疏收益和负载均衡收益降低。
2. **硬件依赖性**：实验在H20（NVLink）上进行，通信带宽充足；在PCIe连接、计算强但通信弱的设备上，P2P迁移增益可能受限，策略需重新校准。
3. **并行策略局限**：主要验证在Ulysses序列并行下；未来可探索在Ring Attention、USP等其他并行模式下的适用性。
4. **评估覆盖**：Animate仅用20个内部测试用例，相对较少；T2V的VBench分数较低（81.6%），可能是因为步蒸馏带来的质量损失，但非FVAttn直接导致。
5. **无训练时稀疏**：论文仅关注推理阶段，未探讨训练时的稀疏注意力，扩展性有待验证。
6. **安全性和冗余**：移动预算（20%头部）和SASA参数（\(n_1=0.8, n_2=0.07\)）可能需针对特定硬件或模型调整，最优值依赖经验设定。
