## 摘要
我们提出MoVerse，一种实时视频世界模型，能够从单张窄视场图像创建可交互导航的场景。这一设置具有挑战性，因为输入仅观测到环境的极小部分，而交互式漫游需要完整的周围世界、持久几何、可控相机运动以及时间一致的高保真观测。MoVerse通过将世界构建与观测渲染分离来解决这一问题。它首先利用拓扑感知扩散将输入扩展为重力对齐的360°全景图，在3D推理前填补缺失的视场。然后，通过全景几何感知残差预测将全景图提升为持久3D高斯支架，生成密集且可直接渲染的空间记忆。最后，高斯条件视频渲染器将沿用户指定相机轨迹的支架渲染结果转换为逼真的视频。为使该渲染器适用于交互，我们训练了一个双向扩散教师模型用于高质量条件渲染，并将其蒸馏为因果自回归学生模型以实现有界延迟流式传输。这种设计结合了显式3D表示的可控性和长程一致性，以及生成式视频模型的感知质量。MoVerse在单个NVIDIA RTX 4090 GPU上支持8 FPS的实时场景漫游，展示了从单张图像创建具有交互视频输出的世界的一条实用路径。

## Abstract
We present MoVerse, a real-time video world model that creates an interactively navigable scene from a single narrow-field-of-view image. This setting is challenging because the input observes only a small fraction of the environment, while interactive roaming requires a complete surrounding world, persistent geometry, controllable camera motion, and temporally coherent high-fidelity observations. MoVerse addresses this problem by separating world construction from observation rendering. It first expands the input into a gravity-aligned 360$^\circ$ panorama with topology-aware diffusion, closing the missing field of view before 3D reasoning. It then lifts the panorama into a persistent 3D Gaussian scaffold using panoramic geometry-aware residual prediction, yielding a dense and directly renderable spatial memory. Finally, a Gaussian-conditioned video renderer translates scaffold renderings along user-specified camera trajectories into photorealistic video. To make this renderer practical for interaction, we train a bidirectional diffusion teacher for high-quality conditional rendering and distill it into a causal autoregressive student for bounded-latency streaming. This design combines the controllability and long-range consistency of explicit 3D representations with the perceptual quality of generative video models. MoVerse supports real-time scene roaming at 8~FPS on a single NVIDIA RTX~4090 GPU, demonstrating a practical path toward single-image world creation with interactive video output.

### 论文核心问题与整体含义（研究动机和背景）

- **核心问题**：从单张窄视场（NFOV）图像创建可交互导航的3D视频世界。输入仅覆盖场景极小部分，而交互式漫游需要完整的360°环境、持久几何、可控相机运动以及时间一致的高保真视频输出。这一问题本质上是欠约束的：必须补全缺失视场、将其转化为相机可控的表示，并实时渲染高质量视频。
- **现有方法不足**：显式3D方法（如点云、网格、3DGS）提供持久几何和准确相机控制，但单图推断时大量区域需靠弱证据生成，易产生孔洞、浮点、深度误差。隐式视频/世界模型能生成视觉上强的视频，但长程几何依赖隐式记忆，在大轨迹或重访区域会出现漂移和身份变化。混合系统使用显式几何作为生成渲染的条件，但稀疏或粗略的锚点引导不足，强生成渲染器往往成本太高或无法实时流式。
- **MoVerse的解决思路**：将**世界构建**与**观测渲染**分离：先完成全景图，再构建持久3D高斯支架，最后用轻量级自回归扩散模型将支架渲染结果转化为高质量视频，实现实时交互。

### 论文提出的方法论：核心思想、关键技术细节

论文提出三阶段流水线：

1. **Stage I: 全景生成（I → P）**  
   - 将未定位的NFOV输入通过**可微自动调平**投影到重力对齐的规范全景空间，消除俯仰和翻滚。  
   - 使用**掩码潜扩散补全**，结合**圆形潜编码**（水平卷积wrap-around）和**移位等变生成**（shift-consistency loss），保持ERP全景的水平S1拓扑闭合，避免人工接缝。  
   - 训练数据**Horizon360**：规范化的室内外全景数据集，将地平线对齐ERP赤道，并采用**偏航中心**约定（输入视图位于全景中心）。  
   - 损失函数：标准潜扩散损失 + 移位一致性损失 + 光流辅助损失。

2. **Stage II: 全景高斯支架构建（P → G）**  
   - 从全景图估计深度，在球形网格上初始化高斯原语：球面反投影得到中心，纬度感知缩放（与cosφ成正比）。  
   - 基于预训练视觉骨干提取多尺度特征，使用**深度引导残差预测**在**角度-逆深度空间**中预测高斯残差（位置偏移、尺度、旋转、颜色、不透明度）。残差头零初始化，保证训练从物理合理的初始化开始。  
   - 训练在HM3D数据集上进行，通过可微渲染优化：目标视角重建损失（L1+LPIPS+SSIM）、源视角重建损失、逆深度损失、角度-深度残差正则化。

3. **Stage III: 高斯条件视频渲染（ˆV → V）**  
   - 先训练一个**双向条件教师**（初始化自Wan2.1-T2V-1.3B），将高斯渲染视频作为密集条件，共享旋转位置编码（Shared-RoPE），条件与目标对齐。教师通过流匹配目标训练。  
   - 然后通过**自强制**（Self Forcing）和**分布匹配蒸馏**（DMD/RAVEN）将教师蒸馏为**因果自回归学生**。学生生成一帧块（4像素帧），条件于当前高斯渲染块，并通过本地KV缓存（MemRoPE：1个sink帧+1个EMA记忆token+滑动窗口3帧）注意已生成的历史，实现有界延迟流式。  
   - 训练数据采用**课程混合**：轨迹对齐几何代理 → 显式高斯渲染 → 全景条件高斯渲染，逐步贴近部署环境。  
   - 部署时使用TAEHV解码器加速，两步去噪，在单张RTX 4090上达到8 FPS。
