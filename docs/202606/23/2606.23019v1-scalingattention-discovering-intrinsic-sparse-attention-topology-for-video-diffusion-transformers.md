## 摘要
虽然扩散Transformer（DiTs）革新了高保真视频生成，但其对3D全注意力的依赖造成了二次计算瓶颈。现有的稀疏方法面临两难困境：动态剪枝带来难以承受的运行开销和内存碎片化，而静态启发式方法无法捕获细粒度依赖。在本文中，我们提出ScalingAttention，一种无需训练的框架，基于一个关键归纳偏置：虽然单个激活依赖于输入，但每个注意力头的高概率区域会迅速收敛到一个稳定的、与提示无关的内在稀疏拓扑。该拓扑是权重编码的、尺度不变的，且易于提取。ScalingAttention通过以下方式将拓扑发现与稀疏控制解耦：（1）WEST（权重编码稀疏拓扑），离线提取鲁棒的块稀疏先验掩码以消除运行时搜索；（2）FAST（保真度感知灵敏度调优），基于扩散保真度要求自适应调整头级稀疏度。为确保实际加速，我们协同设计了硬件对齐的逐位块稀疏内核。在Wan2.1上的实验表明，实现了高达1.90倍的端到端加速，同时保持卓越保真度，在现有最先进基线上建立了新的帕累托前沿。

## Abstract
While Diffusion Transformers (DiTs) have revolutionized high-fidelity video generation, their reliance on 3D full attention creates a quadratic computational bottleneck. Existing sparse methods face a dilemma: dynamic pruning suffers from prohibitive runtime overhead and memory fragmentation, while static heuristics fail to capture fine-grained dependencies. In this work, we propose ScalingAttention, a training-free framework grounded in a key inductive bias: while individual activations are input-dependent, the high-mass attention regions for each head rapidly converge to a stable, prompt-agnostic Intrinsic Sparse Topology. This topology is weight-encoded, scale-invariant, and efficient to extract. ScalingAttention decouples topology discovery from sparsity control via: (1) WEST (Weight-Encoded Sparse Topology), which extracts a robust block-sparse prior mask offline to eliminate runtime search; (2) FAST (Fidelity-Aware Sensitivity Tuning), which adaptively tunes head-wise sparsity based on diffusion fidelity requirements. To ensure practical acceleration, we co-design a hardware-aligned bit-wise block-sparse kernel. Experiments on Wan2.1 show up to 1.90X end-to-end speedup with superior fidelity, establishing a new Pareto frontier over state-of-the-art baselines.

# 论文总结：ScalingAttention

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：视频扩散Transformer（DiTs）采用3D全注意力机制，计算复杂度随token数二次增长，导致推理速度极慢（例如Wan2.1-14B生成5秒视频需约30分钟）。现有稀疏注意力方法面临两难：
  - **动态方法**（如SVG2）运行时剪枝开销大、内存碎片化，且难以利用Tensor Core。
  - **静态方法**（如Radial）离线固定模式，无法适应视频内容的语义结构，易损伤质量。
- **关键发现**：虽然注意力激活值随输入变化，但每个注意力头的高质量（high-mass）区域在不同输入下迅速收敛到一个**稳定的、与提示无关**的集合，称为**内在稀疏拓扑（Intrinsic Sparse Topology）**。该拓扑是**权重编码**的，且**尺度不变**（低分辨率提取的高密度掩码可复用于高分辨率）。
- **本文含义**：重新定义稀疏加速的两个解耦问题：**在哪里允许注意力**（拓扑，静态）和**可容忍多少稀疏度**（敏感性，动态）。ScalingAttention通过解耦实现高效、保真的加速，无需训练。

## 2. 方法论

### 核心思想
- 假设：稀疏拓扑是模型权重的内在属性，与输入无关；而稀疏敏感性沿扩散过程（timestep）和层深度（layer）变化。
- 框架：ScalingAttention = WEST（离线拓扑提取）+ FAST（自适应稀疏控制）+ crm kernel（硬件对齐内核）。

### 关键技术细节

#### (1) WEST（Weight‑Encoded Sparse Topology）
- **块级聚合**：将注意力矩阵 `A` 划分为 `B×B` 块（B=128），计算块显著性：
  \[
  S_{i}^{(u,v)} = \frac{1}{B^2} \sum_{p \in B_u} \sum_{q \in B_v} A_i^{(p,q)}
  \]
- **阈值图构建**：对每个头，从多个校准prompt中收集各块被激活的最小阈值：
  \[
  T^{(u,v)} = \min \{ p \mid \exists i \in [1,M], (u,v) \in \text{Top-}k(S_i, p) \}
  \]
  阈值图编码完整层次：给定目标密度 `p`，可即时生成掩码 `M_p = I(T ≤ p)`。

#### (2) FAST（Fidelity‑Aware Sensitivity Tuning）
- **保真度度量**：使用**Hellinger距离**替代余弦相似度，避免过早饱和：
  \[
  H(P,Q) = \frac{1}{\sqrt{2}} \sqrt{\sum_i (\sqrt{p_i} - \sqrt{q_i})^2}, \quad F = 1 - H
  \]
- **时空保真度分配**：定义目标保真度函数，反映层和timestep的敏感性：
  \[
  A(l,t) = 1 - \left[ \beta \left( 1 - \frac{l + \omega t}{N_L + \omega N_T} \right)^\gamma + (1-\beta) \right]
  \]
  参数 `β, γ, ω` 通过离线优化以满足全局密度约束。
- **分层采样**：仅从每个注意力图的行块中随机抽取少量行（如16行，约0.05%），即可高精度估计Hellinger距离，大幅降低离线校准成本。

#### (3) crm kernel（Compressed Row Mask 内核）
- **表示**：采用压缩行掩码（CRM），每行用对齐的`uint32`数组存储比特，每个比特代表128×128块是否激活。解码时用硬件位运算（CTZ、bit clear）快速遍历活块。
- **优势**：掩码加载开销极小（256k tokens仅需64次`uint32`加载），与FlashAttention兼容，支持Tensor Core。

### 算法流程（文字说明）
1. **离线阶段（每模型一次）**：用少量校准prompt，对每个注意力头、每个层、每个timestep计算块显著性→构建阈值图。同时分层采样计算保真度‑稀疏度曲线。
2. **给定全局密度约束**：通过`A(l,t)`确定每个头所需保真度阈值，从保真度‑稀疏度曲线查得对应稀疏度，结合阈值图生成静态块掩码。
3. **推理阶段**：将掩码上采样到目标分辨率（保守双线性插值），通过crm内核执行块稀疏注意力。

## 6. 主要结论与发现

1. **内在稀疏拓扑**：Video DiTs的注意力头存在稳定的权重编码稀疏结构，可离线提取并跨分辨率复用（recall>96%）。
2. **FAST有效**：相比静态均匀稀疏，FAST在相同全局密度下PSNR提升4~6 dB（表5），尤其低密度区域更显著。
3. **系统加速**：crm内核在0%稀疏度时开销<10%，随稀疏度增加线性加速（50%稀疏度可获得40–60%延迟降低），实现1.65×~1.90×端到端加速。
4. **新帕累托前沿**：在所有对比密度点，ScalingAttention同时取得最高PSNR/SSIM/LPIPS和最高加速，远超SVG/SVG2。

## 7. 优点

- **训练无关**：无需微调或重训练，直接应用于预训练模型。
- **解耦设计**：拓扑发现与稀疏控制分离，降低复杂性，且允许分别优化。
- **硬件对齐**：128×128块、比特掩码、位运算遍历，充分利用Tensor Core，最小化元数据开销。
- **分辨率可伸缩**：低分辨率提取的阈值图通过保守插值可复用于任意分辨率，存储恒定。
- **强鲁棒性**：少量校准prompt（8~10）即可饱和；单prompt稀疏调优mask IoU>91%；支持大长宽比变化。
- **正交性**：可与蒸馏（FastWan）、token重排（PARO）结合进一步提升。

## 8. 不足与局限

- **粗粒度瓶颈**：128×128块大小在极端稀疏（<40%密度）下导致过多保留，无法精细剪枝，与动态方法存在差距。token重排（PARO）可部分缓解，但非完全解决。
- **静态包络冗余**：WEST取的是所有校准prompt的并集，对于局部动态区域可能保留比动态oracle更多的tokens（但加速仍显著）。
- **离线校准依赖单Prompt**：FAST仅使用一个校准prompt，虽然实验显示鲁棒，但理论上若目标视频分布极偏离校准prompt分布仍有风险。
- **未覆盖训练阶段**：仅针对推理加速，未探索稀疏拓扑在训练中的演化或联合训练。
- **硬件特定**：crm内核针对NVIDIA GPU的Tensor Core和位运算优化，推广到其他架构可能需要适配。
- **时间范围**：文中最多处理81帧（Wan2.1-14B 3600 tokens），未评估数千帧超长视频场景。
