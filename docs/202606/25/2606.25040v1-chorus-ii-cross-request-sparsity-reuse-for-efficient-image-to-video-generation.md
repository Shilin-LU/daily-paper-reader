## 摘要
图像到视频生成的扩散模型服务计算成本高，对大规模部署构成了重大挑战。实际I2V工作负载通常包含相似的请求，例如重复的效果模板、相关主体和重复的镜头布局。现有的跨请求加速方法主要通过特征重用来利用这种冗余。我们观察到相似的I2V请求也具有高度一致的稀疏注意力模式，使得历史稀疏掩码可以作为请求条件先验，几乎没有在线掩码预测开销。我们提出了一种以**稀疏性重用**为核心的跨请求重用框架，并以轻量级**指导增强**来保护可选的**特征重用**扩展。我们的稀疏性重用实现为共享稀疏掩码重用，它重用来自类似历史请求的高质量稀疏掩码，以避免每次请求的在线掩码预测。可选的特征重用对高度冗余的时空区域应用降采样计算，在保持效率提升的同时减轻边界伪影。指导增强在重用后加强图像/文本调节，减轻语义漂移和条件遵循问题。实验表明，默认的稀疏性重用配置在保持生成质量的同时实现了**2.16倍**的加速。

## Abstract
Serving diffusion models for image-to-video generation is computationally expensive, posing significant challenges for large-scale deployment. Real I2V workloads often contain similar requests, such as repeated effect templates, related subjects, and recurring shot layouts. Existing cross-request acceleration methods mainly exploit this redundancy through feature reuse. We observe that similar I2V requests also share highly consistent sparse attention patterns, enabling historical sparse masks to serve as request-conditioned priors with almost no online mask-prediction overhead. We propose a cross-request reuse framework centered on \textbf{sparsity reuse}, with \textbf{feature reuse} as an optional extension safeguarded by a lightweight \textbf{guidance enhancement}. Our sparsity reuse is implemented as shared sparse mask reuse, which reuses high-quality sparse masks from similar historical requests to avoid per-request online mask prediction. Optional feature reuse applies downsampled computation to highly redundant spatiotemporal regions, mitigating boundary artifacts while preserving efficiency gains. Guidance enhancement reinforces image/text conditioning after reuse, mitigating semantic drift and condition-adherence issues. Experiments show that default sparsity reuse configuration preserves generation quality with a \textbf{2.16$\times$} speedup.

- **核心问题**：扩散模型用于图像到视频（I2V）生成的计算开销极大，尤其在多步去噪和高分辨率下，在线服务面临高延迟和高GPU成本。现有加速方法多聚焦于单个请求内的冗余（如步间特征缓存、动态稀疏注意力），但在步数高度蒸馏（如4步）的场景中，步间相似性降低，这些方法的开销难以摊销，效果受限。
- **关键洞察**：实际I2V工作负载中常包含大量语义相似的请求（如重复模板、相似主体、相同镜头布局）。这些相似请求不仅共享视觉特征，还表现出高度一致的**稀疏注意力模式**——即使请求不完全相同，其块级稀疏掩码的IoU可达85%以上。这为跨请求重用提供了新的冗余维度。
- **研究含义**：通过跨请求共享历史高质量稀疏掩码，可以几乎消除每次请求的在线掩码预测开销，同时保持掩码的请求自适应性，从而在低步数设定下获得更优的速度–质量权衡。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：以**稀疏性重用**为主路径，辅以可选的**降采样特征重用**和**轻量级引导增强**，构建跨请求加速框架。
- **关键技术细节**：
  1. **稀疏掩码缓存与检索**：
     - 对每个历史请求，在注意力计算时（密集或高精度稀疏）记录其块级稀疏掩码（top‑p + 最小top‑k保证），存入向量数据库，索引为文本和图像嵌入的联合编码。
     - 新请求到来时，进行k近邻检索；若最高相似度超过阈值τ（实验中设为0.4），则直接加载该历史掩码用于当前请求的块稀疏注意力。
  2. **安全回退机制**：
     - **块对访问刷新**：在连续N层（如8层）为一个组，记录哪些块对已被访问，最后一层强制激活未被访问的块对，确保每组内所有Q/K块至少计算一次。
     - **最小top‑k保证**：对注意力分布极尖的头部，强制保留至少k_min个关键块，防止交叉请求分布偏移导致信息丢失。
  3. **头级负载均衡调度**：
     - 由于各头部稀疏度不同，在Ulysses序列并行下会导致GPU负载不均。利用已知的稀疏度分布，通过贪心算法在all‑to‑all之前重新分配头部，使负载方差降低（max/avg从1.34降至1.12），在8GPU上获得约7.28%端到端加速。
  4. **降采样特征重用（可选）**：
     - 在特征缓存（类似Chorus）基础上，不再完全跳过非计算区域，而是对其进行时空降采样后与活跃区域拼接送入注意力层，从而保持全局感受野，消除边界撕裂伪影。
  5. **引导增强**：
     - **首帧键放大**：在自注意力的QKV投影后，将首帧token的K乘以步长相关因子β_step，使后续帧更关注首帧信息。
     - **图像嵌入放大**：在patch嵌入阶段，将图像条件分支的卷积权重乘上α_step再与噪隐层嵌入相加。两者均在重用开始后激活，并随步数衰减。
