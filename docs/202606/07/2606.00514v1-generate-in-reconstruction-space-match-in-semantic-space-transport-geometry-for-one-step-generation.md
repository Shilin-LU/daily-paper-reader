## 摘要
生成建模与自监督表示学习（SSL）优化了结构上不同的目标：生成训练奖励分布保真度，而SSL奖励语义连贯性。然而，近期工作反复发现SSL特征能改善生成训练，尽管这种协同作用的机制尚不清楚。本文在一步生成框架下研究SSL在生成建模中的益处，其中表示的作用是显式的：使用冻结的SSL特征将生成样本与真实数据进行匹配。我们在该特征空间中使用Sinkhorn散度，为Wasserstein距离提供了一个可处理的替代指标，该距离是由Fr\'echet风格评估指标（如FID）近似的人群级差异。我们发现，当在语义结构化的SSL特征空间中计算时，该目标变得非常有效（ImageNet FID降低39倍）。我们将这种行为主要归因于匹配估计：抑制干扰重构细节的语义SSL特征诱导出更紧凑的几何结构，使分布匹配更易处理。因此，最佳训练SSL特征不必与评估指标使用的特征相匹配。特别地，我们展示了使用Inception作为特征提取器可以改善FID，同时降低匹配稳定性和样本质量，揭示了一种度量黑客形式。通过在ImageNet上的大量实验，我们识别出哪些SSL特征家族能带来最佳生成性能，并表明匹配稳定性是选择它们的定量标准。代码可在https://github.com/Genentech/semantic-transport-generation获取。

## Abstract
Generative modeling and self-supervised representation learning (SSL) optimize structurally different objectives: generative training rewards distributional fidelity, while SSL rewards semantic coherence. Yet recent work repeatedly finds that SSL features improve generative training, though the mechanism of this synergy remains unclear. Here, we study the benefits of SSL in generative modeling in the framework of one-step generation where the role of representation is explicit: frozen SSL features are used to match generated samples to real data. We use the Sinkhorn divergence in that feature space, providing a tractable surrogate for the Wasserstein distance, the population-level discrepancy approximated by Fréchet-style evaluation metrics (such as FID). We find that this objective becomes highly effective when computed in a semantically structured SSL feature space (a 39$\times$ reduction in ImageNet FID). We trace this behavior primarily to matching estimation: semantic SSL features that suppress nuisance reconstruction details induce a more compact geometry, making distribution matching more tractable. As a consequence, the best training SSL features need not match the features used by the evaluation metric. In particular, we show that using Inception as the feature extractor can improve FID while degrading matching stability and sample quality, revealing a form of metric hacking. Using extensive experiments on ImageNet, we identify which SSL feature families lead to best generation performance and show that matching stability is a quantitative criterion for selecting them. Code is available at https://github.com/Genentech/semantic-transport-generation.

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- **分离生成空间与匹配空间**：生成在**重构空间（reconstruction space）**（如VAE潜在空间）中进行，而分布匹配损失在**语义空间（semantic space）**（由冻结的SSL特征提取器定义）中计算。这种分离使生成更高效，匹配更稳定。

### 关键技术细节
1. **生成空间**：使用预训练VAE（如SD-VAE）的潜在空间 $\mathcal{M} \cong \mathbb{R}^{4 \times 32 \times 32}$，包含编码器 $\text{Enc}:\mathcal{X}\to\mathcal{M}$ 和解码器 $\text{Dec}:\mathcal{M}\to\mathcal{X}$。生成器 $g_\theta$ 将噪声映射到潜在空间，再解码为图像。
2. **匹配空间**：使用冻结的SSL特征提取器 $\phi:\mathcal{M}\to\mathcal{F}$，直接在潜在空间上提取特征（避免解码到像素域）。分布匹配使用**Sinkhorn散度** $S_\varepsilon(\hat{q}_{\phi,g},\hat{r}_\phi)$ 作为Wasserstein距离的可处理近似。
3. **Sinkhorn公式**：
   \[
   S_\varepsilon(\alpha,\beta) = \text{OT}_\varepsilon(\alpha,\beta) - \frac{1}{2}\text{OT}_\varepsilon(\alpha,\alpha) - \frac{1}{2}\text{OT}_\varepsilon(\beta,\beta)
   \]
   其中 $\text{OT}_\varepsilon$ 是带熵正则化的最优传输问题。该散度消除熵偏差，且收敛到Wasserstein距离。
4. **分类器无导引导（CFG）训练目标**：
   \[
   \min_\theta\;(1+w)S_\varepsilon(\hat{q}_{\phi,\theta}(\cdot|c,w),\hat{r}_c^\phi) - w S_\varepsilon(\hat{q}_{\phi,\theta}(\cdot|c,w),\hat{r}_u^\phi)
   \]
   其中 $w$ 是引导权重，$\hat{r}_c^\phi$ 是条件真实分布，$\hat{r}_u^\phi$ 是无条件真实分布。梯度通过**熵正则化重心的组合** $\hat{T}_\varepsilon^{\text{cfg}}$ 计算，无需反传通过Sinkhorn求解器。
5. **匹配估计稳定性指标 $\tilde{D}_N$**：衡量不同minibatch下Sinkhorn重心估计的差异，用于预测生成质量。
   \[
   \tilde{D}_N = \frac{1}{C_\phi} \cdot \frac{1}{N}\sum_{i=1}^N \|\hat{T}_{\varepsilon,A}(h_i)-\hat{T}_{\varepsilon,B}(h_i)\|^2
   \]
6. **特征提取器训练**：直接训练MAE（不同掩码率）在VAE潜在空间上，或通过蒸馏DINOv3/Inception到该空间。

### 算法流程（文字描述）
1. 采样噪声 $u_0\sim p_0$，生成器 $g_\theta$ 输出 $u_1 = g_\theta(u_0)$。
2. 从真实数据 $x$ 编码得到 $u_r = \text{Enc}(x)$。
3. 分别对 $u_1$ 和 $u_r$ 应用冻结的 $\phi$ 得到特征 $h$ 和 $h_r$。
4. 在特征空间上计算三个Sinkhorn问题：条件交叉、无条件交叉、自传输，组合得到引导更新方向。

## 6. 论文的主要结论与发现

1. **SSL特征是一步生成的关键**：无特征器时FID=134.56，使用SSL特征后降至3.46–6.66（降39×），几乎追上多步扩散模型。
2. **匹配稳定性是预测生成质量的主要指标**：$\tilde{D}_N$ 能直接预测不同特征器的性能排名，有效秩次之。稳定性甚至比几何对齐更重要。
3. **训练特征不必匹配评估特征**：使用Inception特征训练虽能改善FID（几何对齐高），但样本质量反而较差（metric hacking）；而MAE特征尽管对齐度低，但匹配稳定性高，生成质量更好。
4. **Metric hacking效应**：当训练与评估使用同一特征空间时（如Inception），模型可能针对该空间过拟合，FID好但视觉质量差，暴露了单一指标的缺陷。
5. **几何对齐与匹配稳定是两个正交维度**：两者均影响最终性能，但稳定性的贡献更显著。
6. **方法可推广**：在单细胞RNA-Seq上验证了类似规律，PCA作为匹配空间性能最佳。
